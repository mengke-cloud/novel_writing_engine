# 15_testing.md

## 0. 定位

本模块定义引擎变更的测试门禁。适用于脚本、模板、状态规则、Agent 路由和模块工作流更新。

## 1. 执行

```text
python scripts/test_engine.py
```

发布候选版本同时生成报告：

```text
python scripts/test_engine.py --report .agent-test-report.md
```

## 2. 必测层级

```text
单元行为：
  单个脚本的输入、输出、退出码和保护规则。

损坏项目：
  缺失文件、非法状态、重复 ID、占位正文和错误承诺。

兼容性：
  旧版本状态字段修复及现有项目检测。

流程：
  初始化、上下文、检查、归档和状态更新。

发布：
  全脚本编译、完整测试套件、版本与引用审计。
```

## 3. 通过标准

- 所有 Python 脚本编译成功。
- 自动化测试全部通过。
- 测试不访问网络、不修改真实用户项目。
- 失败路径验证默认不覆盖、不归档、不推进状态。

## 4. 强制规则

测试失败时不得提升版本或发布。不得通过删除测试、降低断言或忽略非零退出码掩盖问题。

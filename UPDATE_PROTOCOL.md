\# UPDATE\_PROTOCOL.md



\## 1. 文件定位



本文件用于规范 `novel\_writing\_engine` 的后续更新方式。



本 Skill 是一个长期迭代的小说写作系统，不要求一次完成全部规则。



后续用户在创作过程中发现新的偏好、规则、流程、问题、题材经验、平台经验或写作方法时，都可以通过本文件规定的方式更新 Skill。



\---



\## 2. 更新总原则



每次更新必须遵守以下原则：



```text

1\. 不推翻已确认设定，除非用户明确要求。

2\. 不覆盖用户长期偏好，除非用户明确要求修改。

3\. 新规则必须归入明确模块。

4\. 新规则必须说明是新增、替换、删除还是废弃。

5\. 涉及小说设定的新内容，必须先让用户确认。

6\. 重要更新必须写入 CHANGELOG.md。

7\. 长期偏好必须写入 project\_rules/user\_preferences.md。

```



\---



\## 3. 更新分类



每次更新必须归入以下类型之一。



\---



\### 3.1 新增规则



用于增加新的创作规则、判断标准、输出模板或写作方法。



示例：



```text

动作：新增

规则内容：每章结尾必须留下明确追读点，不能平淡收尾。

```



\---



\### 3.2 替换规则



用于用新规则替换旧规则。



示例：



```text

动作：替换

规则内容：将“默认第三人称写作”替换为“默认第三人称，但用户提供第一人称文本时保持第一人称”。

```



\---



\### 3.3 删除规则



用于彻底删除不再需要的规则。



示例：



```text

动作：删除

规则内容：删除“所有小说默认男频爽文风”。

```



删除规则必须谨慎。



除非用户明确要求删除，否则优先使用“废弃”。



\---



\### 3.4 废弃规则



用于保留旧规则记录，但标记为不再推荐使用。



示例：



```text

动作：废弃

规则内容：废弃“开篇必须先写世界观背景介绍”的旧规则。

```



废弃规则应标记为：



```text

Deprecated / 已废弃

```



\---



\### 3.5 长期偏好



用于记录用户长期稳定的创作偏好。



示例：



```text

用户偏好：用户更需要以小说写作为核心的 Skill，而不是偏影视剧本或分镜。

```



长期偏好应写入：



```text

project\_rules/user\_preferences.md

```



\---



\## 4. 用户更新格式



用户后续可以使用以下格式要求更新：



```text

【更新小说 Skill】

模块：

动作：

规则内容：

原因：

优先级：

```



字段说明：



```text

模块：要更新的模块，例如世界观、大纲、人物、章节、正文、改稿、追读。

动作：新增 / 替换 / 删除 / 废弃。

规则内容：具体要加入或修改的规则。

原因：为什么要这样改。

优先级：强制 / 建议 / 可选。

```



\---



\## 5. 模块名称标准



更新时，模块名称建议使用以下标准名称：



```text

世界观模块：modules/01\_worldbuilding.md

大纲模块：modules/02\_outline.md

人物模块：modules/03\_character.md

章节模块：modules/04\_chapter\_design.md

正文模块：modules/05\_prose\_writing.md

改稿模块：modules/06\_revision.md

追读模块：modules/07\_reader\_retention.md

主控规则：SKILL.md

长期偏好：project\_rules/user\_preferences.md

更新日志：CHANGELOG.md

```



\---



\## 6. 更新优先级



每条规则必须标记优先级。



\### 6.1 强制



必须执行，不得跳过。



适用于：



```text

1\. 世界观确认

2\. 自动生成设定确认

3\. 不擅自改变用户确认设定

4\. 不擅自进入正文

5\. 用户明确要求的硬性规则

```



\---



\### 6.2 建议



默认执行，但可以根据具体任务调整。



适用于：



```text

1\. 文风偏好

2\. 章节节奏建议

3\. 爽点布局建议

4\. 输出格式建议

```



\---



\### 6.3 可选



只在合适时使用。



适用于：



```text

1\. 某些类型小说技巧

2\. 某些特殊章尾钩子

3\. 某些文风变化

4\. 某些额外分析维度

```



\---



\## 7. 更新处理流程



收到用户更新请求后，按以下流程处理：



```text

1\. 判断更新属于哪个模块。

2\. 判断动作类型：新增 / 替换 / 删除 / 废弃。

3\. 判断优先级：强制 / 建议 / 可选。

4\. 检查是否与旧规则冲突。

5\. 如果冲突，按优先级处理。

6\. 输出应该修改的文件名。

7\. 输出可直接复制的新规则文本。

8\. 同步输出 CHANGELOG.md 更新内容。

9\. 如果是长期偏好，输出 user\_preferences.md 更新内容。

```



\---



\## 8. 冲突处理规则



当新规则与旧规则冲突时，按以下优先级处理：



```text

1\. 用户当前明确要求

2\. 用户已确认设定

3\. 用户长期偏好

4\. 当前项目规则

5\. 类型小说规则

6\. 通用小说规则

7\. 系统自动补全建议

```



系统自动补全建议永远不能覆盖用户已确认设定。



\---



\## 9. 设定类更新规则



任何涉及小说设定的更新，都必须经过确认。



设定类内容包括：



```text

1\. 世界观规则

2\. 时代背景

3\. 地图结构

4\. 社会结构

5\. 权力体系

6\. 阶层关系

7\. 修炼体系

8\. 能力体系

9\. 系统规则

10\. 主角身份

11\. 主角目标

12\. 主角金手指

13\. 反派设定

14\. 势力关系

15\. 关键历史事件

16\. 重要人物关系

17\. 核心矛盾

18\. 长线反转

19\. 章节主线方向

```



如果这些内容由系统自动生成，必须标记为：



```text

【系统待确认】

```



用户确认后才能转为：



```text

【用户已确认】

```



\---



\## 10. 确认流程



系统自动生成设定后，必须输出：



```text

以上为系统自动补全设定，尚未确认。



请确认：

A. 全部确认，继续生成

B. 部分修改，我会按你的修改调整

C. 推翻重建，重新生成设定

```



用户未确认前，不得继续生成后续内容。



\---



\## 11. 回滚规则



当用户说：



```text

回滚上一次更新

```



应恢复到上一个版本。



当用户说：



```text

恢复到 0.1.0

```



应恢复到指定版本。



当用户说：



```text

废弃这条规则

```



不直接删除规则，而是将其标记为：



```text

Deprecated / 已废弃

```



除非用户明确说“删除”。



\---



\## 12. 版本号规则



采用语义化版本号：



```text

主版本号.次版本号.修订号

```



示例：



```text

0.1.0

0.1.1

0.2.0

1.0.0

```



\---



\### 12.1 修订号更新



小规则、小模板、小措辞调整时更新修订号。



示例：



```text

0.1.2 → 0.1.3

```



\---



\### 12.2 次版本号更新



新增模块、新能力、新流程时更新次版本号。



示例：



```text

0.1.3 → 0.2.0

```



\---



\### 12.3 主版本号更新



结构性重构、生产稳定版、大规模升级时更新主版本号。



示例：



```text

0.9.0 → 1.0.0

```



\---



\## 13. CHANGELOG 更新格式



每次重要更新都必须写入 `CHANGELOG.md`。



格式如下：



```text

\## \[版本号] - YYYY-MM-DD



\### Added

\- 新增内容。



\### Changed

\- 修改内容。



\### Removed

\- 删除内容。



\### Deprecated

\- 废弃内容。



\### Reason

\- 更新原因。



\### Impact

\- 影响范围。

```



\---



\## 14. user\_preferences 更新格式



长期偏好写入：



```text

project\_rules/user\_preferences.md

```



格式如下：



```text

\## 用户长期偏好



\### 小说 Skill 核心偏好

\- 用户更需要以小说写作为核心的 Skill，而不是偏影视剧本或分镜。



\### 世界观确认偏好

\- 小说 Skill 在运行前需要用户输入世界观，或者帮助用户创建世界观。

\- 系统自动生成的设定必须先让用户确认，确认后才能继续生成后续内容。



\### 写作流程偏好

\- 先确认世界观，再进入人物、大纲、章节或正文。

```

\## 15. 常用更新示例



\### 15.1 更新世界观规则



```text

【更新小说 Skill】

模块：世界观

动作：新增

规则内容：系统自动生成的世界观设定必须先交给用户确认，用户确认后才能进入大纲或正文。

原因：避免设定漂移。

优先级：强制

```



\---



\### 15.2 更新章节规则



```text

【更新小说 Skill】

模块：章节

动作：新增

规则内容：每章结尾必须留下明确追读点，不能平淡收尾。

原因：提高连载追读。

优先级：强制

```



\---



\### 15.3 更新正文风格



```text

【更新小说 Skill】

模块：正文

动作：新增

规则内容：正文默认采用手机阅读友好的短段落，关键情绪和反转单独成段。

原因：提高阅读节奏。

优先级：建议

```



\---



\### 15.4 更新改稿规则



```text

【更新小说 Skill】

模块：改稿

动作：新增

规则内容：用户没有明确要求大改时，只做小修或中修，不擅自重构剧情。

原因：保护用户原文。

优先级：强制

```



\---



\## 16. 禁止事项



更新协议禁止：



```text

1\. 不记录版本就大规模改规则。

2\. 不说明原因就替换旧规则。

3\. 未经用户确认就把系统生成设定写成正式设定。

4\. 用新规则覆盖用户已确认设定。

5\. 把长期偏好写进临时项目规则。

6\. 把临时项目规则误认为长期偏好。

7\. 删除规则时不保留记录。

8\. 忽略 CHANGELOG.md。

```



\---



\## 17. 更新协议核心原则



这个 Skill 应该像一个长期小说创作系统一样维护。



每次更新都要做到：



```text

规则可追踪。

设定可确认。

偏好可沉淀。

旧版本可回滚。

新经验可复用。

```



最终目标：



```text

让 novel\_writing\_engine 随着用户的创作项目不断进化，而不是每次从零开始。

```

# 类型路由

## 适用范围

用于项目初始化和大纲阶段选择类型知识。

## 核心原则

类型由“读者持续期待什么”决定，而不只由背景元素决定。每个项目指定一个主类型，可有不超过两个辅助类型。

## 执行方法

- 期待力量成长、升级与更大世界：玄幻仙侠。
- 期待现实环境中的身份、事业、财富或关系变化：都市。
- 期待问题求解、证据重组和真相揭示：悬疑。
- 期待亲密关系建立、变化与选择：言情。
- 期待时代制度、权力与历史选择：历史。
- 期待科学技术假设及其后果：科幻。
- 期待灾变生存、资源取舍和秩序重建：末世。
- 期待多规则场景、任务世界和跨副本成长：无限流。
- 期待既有作品人物与世界的另一种可能：同人。
- 混合类型先写清主类型承诺，再说明辅助类型如何增强它。

输出：`主类型 / 辅助类型 / 核心承诺 / 兑现周期 / 禁止偏移方向`。

## 常见错误

- 按“出现法术”直接判断为玄幻。
- 同时选择过多类型，导致节奏承诺互相冲突。
- 中途更换主类型却不调整主线。

## 检查清单

- [ ] 主类型能解释核心卖点。
- [ ] 前三章可兑现一次主类型承诺。
- [ ] 辅助类型不抢走主线目标。
- [ ] 类型选择已由用户确认。

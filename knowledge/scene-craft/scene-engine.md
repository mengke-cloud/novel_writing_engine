# 场景引擎

## 适用范围

用于章纲拆场和正文写作。

## 核心原则

场景从人物带着目标进入开始，以局势发生变化结束。场景必须产生结果，不能只传递背景。

## 执行方法

按 `进入状态 -> 即时目标 -> 阻力 -> 尝试 -> 意外变化 -> 选择 -> 退出状态` 组织。下一场由上一场的结果、代价或新问题触发。

## 常见错误

- 场景开始前先长篇介绍背景。
- 人物谈论双方都知道的信息。
- 场景结束后可以整段删除而不影响后文。
- 转场只写时间过去，没有因果连接。

## 检查清单

- [ ] 视角人物有即时目标。
- [ ] 阻力在场景中具体出现。
- [ ] 结尾状态不同于开头。
- [ ] 下一场具有明确触发原因。

# 章节细纲格式

## 适用范围

用于 `chapter` 阶段规划下一章，作为 `draft` 阶段的直接输入。

## 核心原则

- 一章必须完成一个明确的剧情或人物任务。
- 场景之间依靠因果、选择或信息变化连接。
- 章末必须改变读者对下一章的预期。

## 常见错误

- 只有事件，没有人物目标和阻力。
- 场景可任意调换顺序，缺乏因果。
- 章尾靠强行截断句子制造悬念。
- 细纲写成正文，限制后续发挥。

## 检查清单

- [ ] 本章功能可用一句话说明。
- [ ] 起止状态确实发生变化。
- [ ] 每个场景都产生结果或新问题。
- [ ] 章尾钩子承诺了可兑现的后续内容。
- [ ] 待确认项清空后才允许进入 `draft`。

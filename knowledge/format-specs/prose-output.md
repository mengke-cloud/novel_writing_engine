# 正文输出格式

## 适用范围

用于 `draft` 和 `revision` 阶段输出小说正文。

## 核心原则

- 正文只呈现故事，不夹带创作说明。
- 标题、正文和必要的更新建议应清晰分区。
- 未确认设定不得在正文中写成事实。

## 标准结构

```markdown
# 第 {N} 章 {标题}

{正文}
```

用户要求同时展示检查结果时，正文结束后追加：

```markdown
## 检查摘要
- 连续性：
- 人物一致性：
- 伏笔处理：
- 章尾钩子：

## 更新建议
- 建议更新的记忆文件：
- 变化内容：
- 确认状态：【系统待确认】
```

## 常见错误

- 在正文前解释写作意图。
- 用小标题切碎连续叙事，除非项目文风明确要求。
- 将更新建议直接写入正文。
- 输出未完成句、占位符或“此处略”。

## 检查清单

- [ ] 标题与章节编号正确。
- [ ] 正文满足章纲的核心任务和起止状态。
- [ ] 人物、时间、地点和能力符合项目记忆。
- [ ] 没有把待确认设定写成确定事实。
- [ ] 正文完成去 AI 化检查。
- [ ] 状态或记忆变化已形成独立更新建议。

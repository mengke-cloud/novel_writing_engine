# 测试指南

## 运行全部测试

```text
python scripts/test_engine.py
```

生成 Markdown 报告：

```text
python scripts/test_engine.py --report .agent-test-report.md
```

## 覆盖范围

- 项目初始化、预览、重复初始化保护和目录完整性。
- `new`、`partial`、`existing`、`needs_repair`、`skill_root` 检测状态。
- 状态修复、预览不写入和旧版本字段兼容。
- 质量门禁、损坏项目、非法阶段和缺失台账。
- 时间线、人物状态、伏笔承诺和一致性错误。
- 上下文阶段路由、预算、显式文件优先和知识按需加载。
- 归档错误阶段拒绝、预览、正式归档和状态更新。
- 统一 CLI 参数与退出码转发。
- 初始化到归档的端到端流程。
- 旧项目迁移预览、备份、字段映射、原文件保护和迁移后验证。
- 类型指南结构、索引完整性和上下文自动路由。
- 创作数据分析、示例项目和稳定版发布检查。

## 新增测试规则

1. 每个行为修复至少增加一个能够复现旧问题的测试。
2. 写入测试使用独立临时目录，不依赖用户项目。
3. 测试不得访问网络。
4. 失败命令必须验证非零退出码和未产生危险写入。
5. 新脚本必须通过 `py_compile`。
6. 发布前必须运行完整测试入口。
